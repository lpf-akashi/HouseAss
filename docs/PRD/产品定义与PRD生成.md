## 角色设定
你现在是我的产品合伙人，一位经验丰富的互联网产品经理。
## 背景
基于我们之前的市场调研，我决定做一款名为“豪斯助手”的产品，形式暂定为网页或小程序。我的核心价值观主张是【让用户大大减少获取有效房源信息的时间，提升他们的决策效率】
## 任务
请帮我梳理并输出一份简版的PRD文档。需要包含：
1. **产品定位**：用精炼的语言描述我们是谁，服务谁，解决什么核心问题。
2. **核心功能模块**：列出P0级（最高优先级）的核心功能清单，并简述每个功能的预期用户行为。
3. **用户路径**：描述一个典型用户完成一次核心体验（例如：完成一次背诵复习）的操作流程。
4. **非功能需求**：考虑到我是一个独立开发者，请建议一些关于性能、安全性的基本原则。
